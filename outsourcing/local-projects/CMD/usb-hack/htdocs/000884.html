On Thu, 1 Sep 2005, Vandenbroucke Sander wrote:

> I have a question abaout calling a userspace function (static function in OROCOS) from kernelspace (ISR).
> I see there has been an attempt in the canpie driver shipped with OROCOS but it isn't complete.
>
> Is it possible to call userspace from kenelspace? Or am I better of using the userspace interrupts supported by the rtai_usi.o module?
>
I would suggest the latter. A user space call from kernelspace doesn't make
much sense, because the context of the call is not well defined. For
example, there is no "libc" available etc. The userspace interrupt
infrastructure was built especially to let an interrupt "propagate" to user
space and be handled there. Much easier than trying to call userspace stuff
from a kernelspace ISR.

But this is more of an RTAI/LXRT question than an Orocos question, so
you're chances of getting a good answer will be higher on their mailinglist :-)
>From what you say I have the impression that you are writing some device
driver stuff, and this is outside of the Orocos source tree. By definition.
